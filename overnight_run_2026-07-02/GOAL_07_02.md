# GOAL_07_02.md — SafeMPPI on a 5×5 obstacle grid (γ aggressiveness/conservativeness)

Standalone SafeMPPI study on a regular obstacle grid, reusing the frozen best config but **mode 1 (plain
Gaussian sampling)**. No flow matching / no expansion here — this is a clean planner demonstration.

## Scene
- **5 m × 5 m** workspace, discretized at **1 m** (gridlines at 0,1,2,3,4,5).
- **16 circular obstacles**, a **4×4** block on the interior vertices `(x,y) ∈ {1,2,3,4}²`, each **radius 0.2 m**
  → edge-to-edge street width `1 − 2·0.2 = 0.6 m`; for a **0.2 m robot** the center corridor along a street
  midline keeps `0.3 − 0.2 = 0.1 m` body clearance ("the robot always has room").
- **Start** = bottom-left `(0,0)`; **Goal** = top-right `(5,5)`. The straight diagonal passes through the
  obstacles at `(1,1),(2,2),(3,3),(4,4)`, so the planner must weave through the streets.

## Planner
- Frozen `best_area_mode4.json` config, **changed to mode 1 = Gaussian sampling**:
  `polytope_area_sampling=False`, `urgency_size_diff=False` (mode-1 urgency).
- **Sensing/barrier range = 2 m** (`barrier_activation_radius=2.0`, was 3.0).
- Everything else kept: DI, `H=10`, `dt=0.1`, `ns=512`, `noise=0.5`, `u_max=2`, `temp=0.1`, warm-start,
  `centroid_gain=0.2`, `safety_margin=0`, `predict_gain=0`.
- Point-planner (`safety_margin=0`); **robot radius 0.2 m** enforced in the collision/success check.

## Deliverables
1. **γ sweep animation** (`figures/grid_gamma_sweep.{mp4,gif}`): γ ∈ {0.1, 0.5, 1.0} run **simultaneously**
   (3 columns), robot drawn as a 0.2 m disk on the gridded map + nominal polytope + accepted rollouts +
   executed path/trail — showing **γ=0.1 conservative (wide berth) vs γ=1.0 aggressive (tight)**.
2. **Success-only trajectory overlay** (`figures/grid_overlay.png`): for γ ∈ **{0.1, 0.2, 0.4, 0.8, 1.0}**,
   generate rollouts over many seeds, **keep only successes** (reach goal ∧ collision-free for the 0.2 m robot),
   and **overlay ~30 successful trajectories per γ** on the map, **colored by γ**.

## Files

- `run_grid.py` — `render_gamma_sweep()` (deliverable 1) + `overlay_trajectories()` (deliverable 2).
Reuses `overnight_run_2026-07-01` (`di_grid_viz`, `dynamics.Env`) and the repo `SafeMPPIAdapter` via `_paths`.

## Final configuration & results
- **robot radius = 0** (point robot). *Why it matters:* collision = center within `r_obs + r_robot`; with a body
  (0.2 m) the free center-corridor is only `1 − 2(r_obs+r_robot)=0.2 m` and the point-planner grazes (needs
  obstacle inflation); with a **point** the corridor is `0.6 m` and the planner == the robot (no body mismatch).
- **Walls = 14 touching circles per edge** (r=0.2, spacing ≤0.4 m) offset 0.2 m outside each boundary, with a
  small opening at the start/goal corners — they **constrain the nominal/verifier polytope** (grey in the plots)
  and, being dense, keep it from **leaking out of the grid**. No penalty term. **Off the grid ⇒ failure.**
- **Offset fix (2026-07-02):** the γ-sweep gif showed a *constant robot↔obstacle offset* + a boundary polytope
  bulging out of the grid. Diagnosed (`diag_offset.py`): (i) it is NOT a relative-velocity effect (`predict_gain=0`;
  the `eta` HOCBF term is only for moving obstacles — the pedestrian gen used the *same* `load_best_config()`);
  (ii) causes were the **0.1 m `PLAN_MARGIN` inflation** (constant gap) and the **sparse 5-circle walls** (0.6 m
  gaps → `H_P(−0.4,−0.4)=+0.71` leaks out). Fix = **`PLAN_MARGIN=0`** (polytope hugs true obstacles) + **14
  touching walls/edge** (`H_P` at out-of-grid edges now −0.79/−0.65). Still 100 % success, min clearance +0.04 m.
- Mode-1 Gaussian could not solve the walled maze at the best-config speed (0/10 — greedy → stalls or the DTCBF
  fallback drives through interior obstacles; see `figures/mode1_collision_evidence.png`). **Fix (user's):**
  **u_max halved (2→1)** [shorter H-step rollouts ⇒ the DTCBF rejects far fewer ⇒ no fallback] **+ sampling
  variance tripled** (`σ 0.5→0.87`) [more escape exploration]. ⇒ **100 % success** (30/30 seeds per γ),
  collision-free and on-grid at every γ.
- **Avg steps-to-goal (dt=0.1 s), the aggressiveness gradient:** γ0.1 197.7±11.7 (19.8 s) · γ0.2 146.8 · γ0.4 130.5
  · γ0.8 127.7 · **γ1.0 125.1±8.0 (12.5 s)** — conservative γ is ~1.6× slower than aggressive γ.

# Part 2 — Safe Flow Expansion (γ-conditioned FM policy + faithful ACTFLOW)

Train a γ-conditioned flow-matching (FM) policy on the SafeMPPI grid data, then apply **ActiveFlowExpansion
(ACTFLOW)** so its *certified generable set* expands until, per γ∈{0.1,0.5,1.0}, **coverage AND validity > 90%**.

## Stage A — data (`stage2_grid_data.py`)
Mode-1 SafeMPPI (u_max=1, tripled var) over 150 seeds/γ, success-only, sliced into per-step windows:
polar grid `[3,16,12]` + `low5` + past-control history `[16,2]` (GRU input) + target window `U[10,2]`.
Counts: γ0.1 **29.8k**, γ0.5 **19.0k**, γ1.0 **19.0k** windows (`dataset/windows_g<γ>.pt`, revisitable).

## Stage B — model (`grid_policy.py: GridGRUFlowPolicy`, ~297k params)
- low-dim (21) = relgoal-vec(2) + vel(2) + **GRU(16)** over past executed controls + γ(1); `E_l` 21→64→48.
- grid **CNN** `E_g`: Conv 3→8→16, AdaptiveAvgPool(4,3), 192→64.  context = `[E_l;E_g]` = **112**.
- **shallower** velocity trunk (depth 2): `[U(20); ctx(112); time(32)]=164→256→256`; `φ_s`=trunk (256); head→20.
- aux safety-decoder (grid reconstruction, weight 0.3). WORLD/axis-aligned frame (grid + right/up coverage).
- Overfit pretrain (150 ep, `stage3_pretrain.py`) sharpens `φ_s`; baseline validity γ0.1 **95%** / γ0.5 **100%** / γ1.0 **95%**; avg-steps 193/125/127 match the aggressiveness gradient. Baseline coverage ~1.6% (diagonal-concentrated).

## Stage D — coverage/validity (`grid_metrics.py`)
- **Coverage** denom = **C(10,5)=252** monotone right/up staircases; a trajectory covers one when VALID and its
  hysteresis-committed crossing word equals that staircase; coverage = cumulative distinct/252 (monotone ↑).
- **Validity(trajectory)** = SOCP-certified (every 10-window, `verifier_polytope`) ∧ in task-space ∧ monotone
  right/up ∧ reaches goal; validity metric = fraction of the 50 deploys valid.

## Stage E — faithful ACTFLOW (`grid_expand.py`, recovered from `overnight_run_today/src/safeflow.py`)
1 iteration = **1 receding-horizon trajectory** (T=250): (Eq-10) refit `σ_t`=GPUncertainty over `φ_s` of the
queried buffer → (Eq-9) **σ-tilted** window pick (policy samples + broad right/up proposals, cheap safety-filtered,
`w=exp((σ−maxσ)/β)`, systematic-resample) → verify each window → a complete valid staircase → `D_pos` (+covered),
else `D_neg`. **UpdateFlow** = signed `g=∇L⁺−α∇L⁻` (α=0) with **demo replay** (anchors validity) + staircase
inverse-freq replay, large valid buffer (no mode forgetting). Every 50 iters: 50 fresh no-tilt deploys → metrics.

## 2 · Metrics v2 (Stage D → D', `grid_metrics2.py`)
- **coverage_cumulative** = old coverage (distinct staircases ever realized by a valid deploy / 252, monotone ↑).
- **coverage_final (NEW)** = distinct staircases realized by valid deploys **within one measurement batch** / 252
  (can decrease; ceiling = n_measure/252 per batch — reported alongside; diversity of the *current* policy).
- **validity2 (NEW, purely window-level)** = every sliding 10-window (stride 2) satisfies
  SOCP-certified ∧ in-task-space ∧ **goal-approach** — where goal-approach replaces monotone+reach-goal:
  with d_t = ‖x_t − goal‖ over the window: `max_t d_t ≤ d_0 + slack` (no retreat) and
  `d_H ≤ d_0 − min(δ, 0.5·d_0)` (net progress; δ calibrated on demo windows, target ≥97% demo pass-rate);
  windows starting inside the reach disk auto-pass. Reach-rate still logged separately for transparency.
- **Measurement rule (3d)**: 25 fresh no-tilt deploys per γ every 200 iters (binomial 1σ ≈ ±10% — validity
  plots carry a ±1σ Wilson band so real drops are distinguishable from sampling jiggle); final winner gets 100/γ.

## 3 · Expansion v2 (Stage E → E', `grid_expand2.py`)
- **Loss — POSITIVE-ONLY (user correction 2026-07-03: remove U_demo ENTIRELY; the demo anchor pulls
  exploration back to the diagonal and suppresses new modes)**: `L = L_CFM(U_pos, c) − α·L_CFM(U_neg, c_neg)`,
  **no aux term, no demo term**. Inverse-frequency class weighting over discovered staircases only. Demo is
  still LOADED for the drift/forgetting PROBES (diagnostic), never for the gradient. No update until the first
  verified positive exists. (`demo_anchor` flag exists, default OFF, purely for reversibility.)
- **"Nice optimizer" for the entangled encoder+field (user: "how to optimize both encoder and the vector
  field")**: two Adam param groups — velocity field (trunk+head) at `lr`, context encoders (E_g/E_l/GRU) at
  `lr·enc_lr_mult` — plus cosine annealing. `enc_lr_mult<1` slows the context map (less drift/forgetting while
  the field still adapts); **`enc_lr_mult=0` == frozen encoder** (the causal freeze test, now a sweep point).
- **Notation (3c)**: `temp` = FM sampling temperature (initial-noise scale, `temp·randn`); `beta` = Eq-9 tilt
  temperature in `w = exp((σ−maxσ)/β)`. Fixed `temp=1.3` (moderate, near the policy's own distribution).
- **Single model, all γ**: training round-robins γ ∈ {0.5, 1.0, 0.1} (one trajectory per iter); measurement
  reports validity2/coverage per **each** γ every 200 iters (catches cross-γ forgetting the moment it starts).
- **var(σ) (metric B)**: per exploration step, the variance of σ across the 64 candidates; averaged per iter
  and per measurement window (the "is the uncertainty signal alive" metric).
- Candidates stay FM-only (N=64, no broad), cheap safety pre-filter on; per-window buffer labels use the cheap
  approach∧taskspace check (SOCP runs once per trajectory — v1 wastefully ran SOCP per step).

## 4 · Entanglement diagnosis + validity sanity tests (no edits to existing code)
Because OOD here entangles control-window AND context (both shift during expansion), every v2 run logs:
- **per-module grad flow during expansion** (same RMS-per-param norms as pretrain): if E_g/E_l/GRU norms spike
  when new positives enter, the update is rewriting the conditioning map, not just the vector field;
- **context drift probe**: 256 frozen demo windows; ‖ctx_t − ctx_0‖/‖ctx_0‖ and cos(ctx_t, ctx_0) per
  measurement — the same physical state drifting in context space = the field chasing a moving target;
- **demo forgetting probe**: val-CFM on 1024 frozen demo windows (identical noise draws) — rises ⇒ losing the
  pretrained safe manifold;
- **GP health**: σ_fresh vs σ_buffer, novelty rank-correlation (D4) — σ signal degrading ⇒ exploration goes
  random ⇒ invalid trajectories ⇒ validity jiggle;
- **causal test — `freeze_enc` arm**: expansion with E_g/E_l/GRU FROZEN (only trunk+head learn, context map
  fixed). Freeze ≥ unfrozen on validity at equal coverage ⇒ context-encoder gradients ARE the harm.
- **validity sanity lever — `pos_margin` arm**: only windows with min obstacle clearance ≥ 0.03 m enter D_pos
  (grazing windows discarded, neither pos nor neg) — trains on comfortably-safe data to lift the validity
  ceiling set by zero-margin grazing.

## 6 · NN uncertainty estimator + round-based fine-tuning — PARALLEL BACKUP plans (`uncertainty_nn.py`)
User 2026-07-03: keep these as **backups run in parallel** to the GP-RBF main path (do not harm it); report
if useful. The paper (ACTFLOW arXiv 2606.08802) uses, for its continuous experiments, a **bootstrapped ensemble
of 5 MLPs** (2 hidden × **64 units — shrunk from the paper's 100** per user, ReLU, 10% dropout; each on a 90%
bootstrap of accumulated (φ_s, y) feature–LABEL pairs, Adam 1e-3, ≤1000 steps; σ = std across members) and a
**round-based fine-tuning** loop (collect a batch → refit estimator → block of gradient steps on the accumulated
valid buffer, warm-up before the first update). Both are SEPARATE modules, positive-only, launched at base:
- **NN vs GP is a real difference, not cosmetic**: the GP σ is pure NOVELTY (distance-to-buffer over queried
  φ_s); the NN is a validity CLASSIFIER over φ_s using BOTH accepted & rejected windows, so its σ (member
  disagreement) peaks at the **decision boundary of the valid set** — informative-query uncertainty, the term
  Eq-9 actually maximizes. That boundary-seeking signal is the more principled unimodal→multimodal driver.
  Toy check (ring/half-space in a 16-D rep): σ_boundary 0.22 ≫ σ_deep 0.00. Cold start σ≡1 (matches GP).
- **Round schedule (scaled small, user-approved)**: round = 16 trajectories → refit estimator → 200 fine-tune
  steps on accumulated valid buffer; first update deferred until 300 valid windows. Warm-start + early-stop NN
  refits (keep weights across refits, ≤1000 steps, stop on plateau) keep it tractable at frequent refits.
- **Wired as constructive-append arms**: S7-unc {gp°, nn}, S8-sched {online°, round}; plus a guaranteed
  `flagship-ref` run (nn+round at the incumbent) so the paper's headline combo always has a data point.
  `grid_expand2.py --unc {gp,nn} --schedule {online,round}`.

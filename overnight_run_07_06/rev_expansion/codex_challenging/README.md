# Challenging-stadium Safe Flow Expansion workspace

The maintained implementation is [`afe_restart/`](afe_restart/). It is the
clean planned-window AFE restart in which the object generated by the flow is
the same H=10 action window submitted to the full verifier and, when safe, the
same object replayed for training.

Start with:

- [`afe_restart/README.md`](afe_restart/README.md) — executable stage map,
  commands, inputs, outputs, and ownership of every framework component;
- [`afe_restart/METHOD.md`](afe_restart/METHOD.md) — scientific contract and
  fault-to-fix audit;
- [`afe_restart/PROGRESS.md`](afe_restart/PROGRESS.md) — immutable experiment
  chronology, including rejected runs and the radius-1.0 diagnostic.

## Directory status

| Path | Status | Purpose |
|---|---|---|
| `afe_restart/` | authoritative | Clean planned-window AFE implementation, tests, evaluation, visualization, and reports. |
| `giant_obstacle_ood/` | historical/diagnostic | Earlier staged experiment and the raw SafeMPPI/polytope diagnostic tools. Do not import its expansion trainer into the clean method. |
| `reference/` | comparator/reference | Kazuki/Mizuta adapter source and the original paper-figure style references. |
| root scripts such as `grid_hp_expt.py` | shared dependency | Policy architecture, scene features, and legacy-compatible loaders used explicitly by the restart. |
| `stage_results/`, `afe_restart/stage_results/` | generated, local | Large datasets, checkpoints, ledgers, videos, and reports. They are intentionally not versioned. |

The generated result trees currently occupy more than 100 GB. Source control
contains the code and contracts needed to reproduce them; each run writes
hash-locked manifests so a result can be audited without committing the bulk
artifacts.

## Current scientific status

The clean implementation contains no `demo_frac`, LwF, data/model anchoring,
easy/frontier curriculum, uncertainty-weighted replay, or negative-sample
objective. Flow acquisition and scientific audits use temperature 1.0;
temperature 0.5 is visualization-only.

The latest radius-1.0 canonical-SafeMPPI diagnostic established that expert
feasibility is not the blocker (14/14 successful expert episodes), but the
three-round clean expansion did not produce a robust closed-loop gain. That
negative result and its diagnostics are recorded in
[`afe_restart/PROGRESS.md`](afe_restart/PROGRESS.md); it must not be hidden by
an ad-hoc rescue mechanism.

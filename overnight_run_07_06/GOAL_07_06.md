## 0. Thesis
Our flow policy conditions the **generator on the scene** (H_P encoder) and learns **multimodal** avoidance
*in the prior*; safe expansion then discovers new modes via **σ-uncertainty** without trading safety for
performance. We contrast with `cfm_mppi` (Kazuki), whose generator is start+goal-only and whose execution is
unimodal (§1). **Central question of 07_06:** *which pretraining data most enables flow expansion?* We answer
it by holding the model + expansion recipe fixed and swapping the pretraining dataset (§3), evaluated by
**success/collision from the origin** (§2).

**Why the OOD-origin test:** off-diagonal data gives the encoder rich *feature/spatial* confidence but the
robot starting at **(0,0)** is **out of distribution** → baseline SR/CR should be *poor*, yet safe expansion
should **discover the diagonal-axis modes the data knows** → multimodal, safety preserved. This isolates the
expansion mechanism from raw in-distribution performance.

  transient in the sampler, discarded at commit.
- **Their metrics** = collision-rate (%), final goal-distance, per-step **compute** time — **no thresholded
  success-rate, no time-to-goal** in code. Our SR is a metric they don't report.
- OT path `x_t=(1−t)x₀+t·x₁` — identical to our φ_s interpolation.
- **Our story:** multimodal + scene-conditioned + *scarce data* + certified, beating their global SR/CR.

## 5. Safe Flow Expansion overhaul  (`grid_expand2.py` — curriculum + smooth updates)
**Defaults (kill instability at the source):** **α = 0** (drop the unbounded negative-unlearning term,
`grid_expand2.py:223-226`), **demo_frac δ = 0**, **lwf_eta η = 0**, **sampling temp = 1** (faithful),
**batch = 64** (from 128, `:81`), slow updates.

### 5a. Positive-first warm-up  [NEW gate]
Collect positives with **no gradient step** until `n_pos ≥ N_warm` (default **1000**; option 2000), then start
training. (Today online mode updates on the *first* positive — `:421-425` — so this is a new gate.)

### 5b. Concrete easy / mid / frontier scoring  [reproducible thresholds]
Per positive window, compute: **σ** (`unc.sigma(phi_s)`), **SOCP-margin** (`grid_metrics2.window_min_clearance`,
`:100-108`), **jerk(U)** = mean‖Δ²U‖, **mono** = fraction of steps decreasing goal-distance. Using
within-buffer quantiles (adaptive, reproducible):
- **easy** = σ ≤ q33(σ) ∧ margin ≥ q67(margin) ∧ jerk ≤ q33(jerk) ∧ mono ≥ 0.9
- **frontier** = σ ≥ q67(σ) ∨ margin ≤ q33(margin)
- **mid** = everything else
Batch composition **70% easy / 30% mid / 0% frontier**, ramping to **33/33/33** (5c).

### 5c. Frontier ramp + cooldown  (closed-loop trigger)
- **β (σ-tilt temp, `grid_rollout.py:149`) anneal 1.0 → 0.5 → 0.2 → 0.1**: high β = faithful/uniform proposal,
  low β = select the high-σ frontier. **Trigger = origin SR/CR** (ramp down a step when SR plateaus / CR stays
  low). Candidates: (i) SR/CR-adaptive [default], (ii) smooth β(t)=1.0·(0.1)^{t/T}, (iii) fixed-step.
- Batch mix 70/30/0 → 33/33/33 synced to the same trigger.
- **Cooldown:** lower lr (×0.3), **1–2 Adam inner steps early** (from `inner_steps=12`, `:80`), balanced buffer.

### 5d. Delete the staircase inverse-frequency weighting  [`grid_expand2.py:187-196`]
Remove `freq = Counter(pos["tag"])` / `w = 1/freq[t]` (sampled via `torch.multinomial` `:206,211`). Replace
with the easy/mid/frontier sampler (5b) so discovery isn't hard-coded toward any staircase region.

## 9. FINDINGS — curriculum expansion module + the forgetting/anchor laws (2026-07-07)
Module: **`grid_expand_cur.py`** (grid_expand2 untouched). Key code: warm-up gate (`run_expand_cur`, n_pos≥warmup_pos
check ~L291) · composite easy/mid/frontier pools (`curriculum_pools` L119-141; strict = top-15% composite —
the 4-way AND starved the pool to ~1%) · stratified batch (`_draw_batch` L144-155) · update + **field/enc
grad-RMS instrumentation** (`update_flow_cur` L158-196) · δ/η anchor = demo-replay + LwF-vs-frozen-teacher
(`update_flow_cur` demo/teacher branches; `_load_demo`) · β schedules incl. smooth exp/aggressive (~L258-265) ·
collapse-termination + best.pt (~L329-339) · labeled buffer-DB every 1k (`_save_viz_db` L213-240 →
`<outdir>/viz_db/it<t>.pt`: 64 pos {grid,low5,U,label,σ,margin,jerk,mono} + 16 neg with criteria_status reason).

**Laws (all code-verified):**
1. **Un-anchored curriculum FORGETS** — repr20 collapses under ANY β (flat/gentle/step: SR .31→.00-.07);
   repr15 peaks (SR .08→**.53**) then decays to .21. Forgetting is in the FIELD; freezing the encoder can't stop it.
2. **δ/η anchor RESCUES + SUSTAINS** — a_r20_delta SR .31→.32 held w/ CR .53→.29 @2k; δ (replay) is the key,
   η-only fails (a_r20_lwf .13). **Over-anchoring hurts**: δ.4/η.1 (a32_unf_hi) final .42/.42 — caps the gain.
3. **Warm-up gathering is unsafe/unguided** — σ flat early (cold GP): ~59% of warm-up deploys collide
   (logged `WARMUP cleared` line).
4. **repr sizing** — repr is the vector-field feature dim; 20 is fragile at depth (a20T/a20E pretrain into a
   0.00-SR OOD-origin basin twice), **repr32/48 robust**. GP σ needs small φ_s (256-d had no kernel spread).

**★ WINNER `a32_unf`** — base `results/hp_repr/pretrained_a32.pt` (repr32: trunk 89→160→96→32, head 32→20,
enc_depth 2, **32×32 H_P** grid — parametrized in `grid_hp_expt.GridHPFlowPolicy(repr_dim, trunk_hidden,
enc_depth, grid_hw)`); expansion **unfrozen enc (lr×0.3, clip 5) · δ0.25/η0.05 · easy-strict · temp1 · α0 ·
batch64 · inner4 · lr1e-4 · 5k** → **SR 0.35→0.73, CR 0.48→0.14**, sustained. Ckpts
`results/sweep_overnight/a32_unf/{best,final,ckpt_*}.pt`. A/C knob sweep around it (results/sweep_ac/):

within per-measure noise (±0.2) baseline≈C2 tied on top. Judge on **mean of last-3 measures + best.pt**.

## 13. rev_expansion — FRESH-ONLY 2-class curriculum + 9-arm OFAT sweep (2026-07-08)
Isolated copy folder **`rev_expansion/`** (parent `*.py` untouched; path shim → local edited `grid_metrics2`).
**Why:** the pile-based loop trains each step on ~60 fresh : 60000 stale windows (eviction is uniform-random
over ALL history) → it keeps learning from the *worse early* policy. **New method: every update uses ONLY this
iter's fresh rollouts**, labeled into 2 classes → batch composition fully known/controllable, and the σ
(GP-uncertainty) threshold becomes an explicit, ablatable knob (paper claim: *uncertainty enables stable
curriculum learning*).

**valid2 FIX** (`rev_expansion/grid_metrics2.py`, dual goal = performant + safe·SOCP): `approach_ok` = **net
progress ONLY**, `d0−dH ≥ min(0.10, 0.5·d0)` — bounded-retreat + per-step dropped (over-restrictive; SOCP owns
safety). Threshold 0.12 → **0.10**.

**Per-window labeling (fresh valids only):** valid = taskspace ∧ SOCP(traj) ∧ net-progress ≥ 0.10.
**frontier** = high-σ (≥ q_hi 0.67 quantile) OR low-margin (≤ q_lo 0.33 quantile) OR **net-progress ≥
prog_floor** (bold-and-safe). **easy** = valid & not frontier. (No mid.)

**Batch / efficiency:** total **batch = 32** = 8 demo (demo_frac 0.25) + 24 fresh at easy:frontier mix
**7:3 → 5:5**, availability-capped, drawn with replacement. **Early-stop gather:** roll out only until the
24-window quota fills — ~**1–2 rollouts/iter** (one rollout yields ~hundreds of valid windows), ~5–8× faster.
**inner 2/4/2** (early/mid/cool) so each window gets several passes. viz_db (labels + σ/margin/prog) every 100 iters.

**Baseline A0** (aligned to a32_unf): rollouts-cap 10 · prog_floor 0.3 · β step (1.0,0.5,0.2,0.1) @ (0,.25,.5,.75)
· early/cool 0.1/0.75 · demo .25 / lwf .05 / easy-strict / unfrozen-enc·0.3 · `results/hp_repr/pretrained_a32.pt`
· 1000 iters · seed 0 · measure_every 100 (M=25).

# External Dataset Recommendations For Safe Flow Expansion

Use this note for GOAL.md stages 1, 2, and 4: scene construction, verified
SafeMPPI rollouts, and gamma-conditioned FM pretraining.

## Recommendation

The best fit is not Mizuta `train80_ego.pt`. That file is ego-only pedestrian
snippets and does not contain obstacle context or robot demonstrations.

For our pipeline, prefer:

1. **NavWareSet** as the first real robot-social-navigation dataset.
   - Has robot poses, participant positions, occupancy grids, robot ROS bags,
     and velocity command topics.
   - Includes HSR and Jackal robots, social and non-social behavior, and
     scenario types such as frontal approach, pedestrian obstruction, blind
     corner, following human, perpendicular traffic, and circular crossing.
   - Directly matches the tensor we want:
     `robot state/control + nearby humans + static obstacles`.
   - Source: https://anr-navware.github.io/navwareset/

2. **THOR-MAGNI** when we want accurate indoor mocap and robot-human interaction.
   - Has human and robot navigation/interaction, position and head orientation
     motion-capture data, maps/goal positions, 3D LiDAR, gaze, and scenarios
     where the robot is a static obstacle or moving agent.
   - Larger and heavier than NavWareSet, but higher quality for controlled
     indoor trajectory analysis.
   - Source: https://zenodo.org/records/13865754

3. **Mizuta UCY/SDD/ETH or trajdata** only for scene-context generation and OOD
   testing, not for robot expert training.
   - These are useful for many dynamic obstacle scenes.
   - Robot expert labels should be generated by deploying SafeMPPI in these
     scenes, then recording `context -> safe control sequence`.
   - Source: https://github.com/NVlabs/trajdata

## Secondary Datasets

- **SCAND**
  - Good for behavior cloning of human-teleoperated robot navigation.
  - Has 8.7 hours, 25 miles, 138 robot trajectories, Jackal and Spot, and
    multi-modal robot sensors.
  - Less ideal if we need clean ground-truth human positions for every nearby
    obstacle.
  - Source: https://www.cs.utexas.edu/~xiao/SCAND/SCAND.html

- **SiT**
  - Robot navigates through crowded indoor/outdoor environments and provides
    LiDAR/camera/RTK plus 2D/3D box, class, and ID annotations.
  - Strong for perception-to-trajectory settings; heavier to convert into our
    low-dimensional SafeMPPI tensors.
  - Source: https://spalaboratory.github.io/SiT/

- **JRDB**
  - Egocentric JackRabbot data with many time-consistent people trajectories,
    360-degree visual/LiDAR sensing, wheel encoders, and indoor/pedestrian-area
    robot scenes.
  - Strong perception benchmark; less direct than NavWareSet for clean
    `robot command + participant xy + occupancy` supervised control data.
  - Source: https://arxiv.org/abs/1910.11792

- **Stanford Drone Dataset**
  - Large top-down outdoor multi-agent trajectories with pedestrians, bicyclists,
    skateboarders, cars, buses, and carts.
  - Useful for OOD dynamic-agent scene generation, especially faster agents.
  - No real robot policy/control.
  - Source: https://cvgl.stanford.edu/projects/uav_data/

## Proposed Use In Our Pipeline

Stage 1 scene context:

- Use NavWareSet first because it already has `robot_x robot_y robot_yaw` plus
  participant positions and occupancy grids in simple pose products.
- Keep Mizuta UCY/SDD as an OOD comparison, but treat the robot labels as
  simulated SafeMPPI rollouts.

Stage 2 verified polytope loop:

- Build finite-sensing obstacles from participant positions.
- Build static obstacle halfspaces/occupancy from the occupancy grid.
- Run SafeMPPI with gamma rejection and record accepted/rejected rollouts.

Stage 4 FM pretraining:

- Do not train on raw logged robot trajectories alone unless the goal is behavior
  cloning.
- For the paper's claim, deploy our own SafeMPPI/verifier over the scene and
  record:
  `start, goal, ego_current, ego_history, action_history,
  nearest_obstacle_history, gamma, safety_margin -> safe control sequence`.

This keeps the expert label aligned with the DTCBF gamma guarantee instead of
mixing in a human demonstrator's unverified style.
